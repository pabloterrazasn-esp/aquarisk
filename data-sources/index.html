Fuentes y acceso a datos

Bases de datos, cartografia y puntos de acceso oficiales

TerraNava publica esta página para dejar visible dónde se obtienen los datos base usados en AquaRisk y en sus productos institucionales. La regla de trabajo es simple: fuente oficial primero, referencia europea cuando aporta valor y fallback abierto trazable cuando no existe alternativa mejor o cuando el caso requiere continuidad metodológica.

Uso de esta página

Acceso, no republicación indiscriminada

Cuando una fuente externa es oficial o de referencia, TerraNava prioriza enlazarla y documentarla. Los archivos pesados y los datos crudos permanecen fuera del frontend; en la web se públican derivados ligeros y metadatos legibles.

Fecha de verificación

Los enlaces de esta página fueron verificados por TerraNava el 11 de marzo de 2026 como puntos de acceso válidos para preparación de capas, inventarios o soporte metodológico.

Regla de publicación

Que pública TerraNava y que mantiene fuera del frontend

Se publica

Derivados ligeros, tiles, subcuencas, históricos por cuenca, fichas de estaciones, metadatos y paneles de cobertura.

No se pública

DEM crudo, mosaicos pesados, shapefiles originales, procesos intermedios y productos experimentales sin ficha metodológica.

Acceso a datos

Cuando una fuente es oficial o abierta y estable, TerraNava enlaza el punto de acceso original para mantener trazabilidad y facilitar auditoria externa.

Principio operativo

Raw offline, derivados online. El laboratorio y el SSD conservan los insumos pesados; la web pública solo aquello que puede sostener una lectura hidrológica clara y prudente.

Siguiente capa

El siguiente salto para España premium es cerrar el inventario de estaciones, preparar el DEM backbone y públicar derivados de relieve con QA y niveles de servicio visibles.

Continuidad metodológica

Las fuentes deben ser legibles antes que decorativas

Esta página existe para que TerraNava y AquaRisk puedan ser auditados desde fuera. La credibilidad no depende solo de tener capas o informes, sino de hacer visible de dónde salen los datos y en qué condiciones deben interpretarse.
