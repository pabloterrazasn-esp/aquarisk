# Hoja de Ruta de Preparacion para Auditoria v1

## Objetivo

Preparar TerraNava y AquaRisk para una revision tecnica e institucional externa, de forma que la cartera visible de servicios pueda defenderse con evidencia, trazabilidad y coherencia entre web, metodologia, datos e informes.

## Paquete minimo de evidencia por linea de servicio

Cada linea visible debe contar con:

1. ficha de servicio
2. nivel de servicio declarado
3. fuentes principales documentadas
4. metodologia resumida
5. ejemplo de informe o salida
6. limites de uso
7. modulo o respaldo AquaRisk, cuando aplique

## Hallazgos previsibles de una auditoria

- sobrepromesa de capacidades no suficientemente documentadas
- falta de separacion entre screening e informe defendible
- capas vistosas sin metadatos visibles
- expansion por paises sin checklist minimo
- lenguaje demasiado fuerte para evidencia aun parcial

## Checklist de preparacion

### A. Coherencia institucional

- la web publica debe presentar la misma cartera corta que los documentos internos
- metodologia y servicios no deben contradecirse
- AquaRisk debe usar el mismo lenguaje de niveles de servicio

### B. Evidencia tecnica

- fuentes accesibles y documentadas
- inventarios por pais
- periodos y resoluciones visibles
- versionado de datos
- informes con trazabilidad

### C. Evidencia cartografica

- figuras reproducibles
- plantillas estables
- leyendas, escalas y disclaimers consistentes
- sin apariencia de precision no respaldada

### D. Evidencia operativa

- checklist de activacion por pais
- QA de capas y figuras
- changelog o checkpoint visible
- responsables de actualizacion internos

## Secuencia de trabajo recomendada

### Paso 1. Ordenar la cartera visible

- adoptar la cartera corta de 5 lineas
- mantener los 6 ejes como marco institucional
- definir tipologias de apoyo por cliente

### Paso 2. Vincular cartera y producto

- mapear cada linea a AquaRisk
- declarar que modulos respaldan cada una
- identificar vacios de datos y QA

### Paso 3. Endurecer la evidencia

- ejemplo de informe por linea
- metodologia resumida por linea
- enlaces publicos a fuentes y acceso a datos
- figuras cartograficas reproducibles

### Paso 4. Preparar el caso premium

- Bolivia consolidado
- España premium como benchmark
- no abrir lineas o paises con evidencia incompleta

## Criterios de aprobacion interna

Una linea puede considerarse lista para auditoria cuando:

- existe ficha de servicio
- existe metodologia asociada
- existe ejemplo de informe
- existe fuente principal o inventario
- existe disclaimer de alcance
- existe respaldo en AquaRisk o una nota explicita de por que aun no lo tiene

## Stop rules

No usar como promesa fuerte:

- servicios sin ejemplo de salida
- servicios sin fuente o metodo visibles
- servicios que dependan de una capa experimental no etiquetada
- servicios que sugieran validacion observacional sin evidencias
